4.1.1 #components.buttons.icon-buttons Icon Buttons

Icon color matches text color Small vertical line between icon and text. Buttons that have “forward” actions have icons to the right of the text, and those with “back” actions have icons to the left.

Example 
<button class="button--icon button--secondary"><img alt="arrow-left" src="../img/i-arrow-left--secondary.svg">Previous</button>
<button class="button--icon button--secondary button--icon--right">Next<img alt="arrow-right" src="../img/i-arrow-right--secondary.svg"></button>
<button class="button--icon button--secondary"><img alt="charts" src="../img/i-chart--secondary.svg">View charts</button>

    4.13 #components.toggles Toggles

    We use radio buttons for toggles, wrapped in a fieldset with a legend to label them. Apply standard button classes to the .button span to style them

    Example
    View by: 
    <fieldset class="toggles">
  <legend class="label">View by:</legend>
  <label for="toggle-state">
    <input id="toggle-state" type="radio" name="receipt-aggregate" value="by-state" checked>
    <span class="button button--primary">State</span>
  </label>
  <label for="toggle-contribution-size">
    <input id="toggle-contribution-size" type="radio" name="receipt-aggregate" value="by-contribution-size">
    <span class="button button--primary">Contribution Size</span>
  </label>
  <label for="toggle-employer">
    <input id="toggle-employer" type="radio" name="receipt-aggregate" value="by-employer">
    <span class="button button--primary">Employer</span>
  </label>
  <label for="toggle-occupation">
    <input id="toggle-occupation" type="radio" name="receipt-aggregate" value="by-occupation">
    <span class="button button--primary">Occupation</span>
  </label>
</fieldset>
